# Frontend Design Improvements

## Problem Statement
The frontend design lacks consistency, proper visual hierarchy, and adherence to modern UI principles. Components use arbitrary spacing, inconsistent typography, fixed heights, and custom implementations where Bits UI components should be used.

## Goals
1. Implement a proper design system with consistent spacing (8-point grid)
2. Establish clear typographic hierarchy
3. Replace fixed heights with responsive, content-based sizing
4. Utilize Bits UI components consistently
5. Improve visual hierarchy and content prioritization
6. Remove inline styles and use CSS variables/classes
7. Fix inconsistent spacing and padding values
8. Improve FeedBox component design
9. Enhance AppHeader with better visual hierarchy
10. Ensure proper responsive breakpoints and mobile-first approach

## Tasks
See tasks.md for detailed task list.

## Acceptance Criteria
- All spacing follows an 8-point grid system
- Typography uses a defined scale with clear hierarchy
- No fixed heights that cut off content
- Bits UI components used where appropriate
- Clear visual hierarchy guides user attention
- Consistent use of CSS classes/variables instead of inline styles
- Responsive design that works across breakpoints
- Improved usability and readability of FeedBox component
- AppHeader has improved visual hierarchy and spacing
